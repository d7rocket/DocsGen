**Helper function: `_add_colored_heading(doc, text, level, color_hex)`**
- Per D-14/D-15 and Pattern 3. NEVER use style mutation.
- Creates heading with `doc.add_heading(level=level)` (no text arg — per Pitfall 7)
- Adds run with `heading.add_run(text)`
- Sets `run.font.color.rgb = RGBColor(r, g, b)` using parse_hex_color
- Sets run.font.bold = True for level 1
- Returns the heading paragraph

**Helper function: `_set_cell_background(cell, color_hex)`**
- Per D-16 and Pattern 4.
- MUST create a NEW parse_xml element for EVERY cell (per Pitfall 3 — never reuse XML elements)
- Strips '#' from color_hex if present
- Creates shading: `parse_xml(f'<w:shd {nsdecls("w")} w:fill="{color_hex_clean}" w:val="clear"/>')`
- Appends to `cell._tc.get_or_add_tcPr()`

**Helper function: `_set_paragraph_shading(paragraph, color_hex)`**
- Per Pattern 5. For code block backgrounds.
- Same pattern as cell background but on `paragraph._element.get_or_add_pPr()`

**Helper function: `_add_paragraph_border(paragraph, color_hex="D0D0D0")`**
- Per Research code example. For code block borders.
- Creates pBdr XML with top/left/bottom/right single borders, sz=4, space=4

**Helper function: `_create_code_style(document)`**
- Per D-19 and Research code example.
- Adds 'Code' style of type WD_STYLE_TYPE.PARAGRAPH
- Base style = 'Normal'
- Font: Courier New, 9pt, color RGBColor(0x33, 0x33, 0x33)
- Returns the style object
- If style already exists (on subsequent calls), return existing

**Helper function: `_build_cover_page(doc, config, primary_color)`**
- Per D-17.
- Uses the document's first section (doc.sections[0])
- Sets `section.different_first_page_header_footer = True` (per D-18)
- Content on cover page:
  1. Add 3 blank paragraphs for top spacing
  2. Client logo centered, large (~3 inches width). Use `paragraph.add_run().add_picture(config["client_logo"], width=Inches(3))`. Paragraph alignment = WD_ALIGN_PARAGRAPH.CENTER.
  3. 2 blank paragraphs spacing
  4. Client name: large (24pt), bold, primary color, centered
  5. Report name: 18pt, primary color, centered
  6. 1 blank paragraph
  7. Version and date: 12pt, grey (RGBColor(0x66, 0x66, 0x66)), centered. Format: f"Version {config['version']} | {date.today().strftime('%B %d, %Y')}"
  8. Multiple blank paragraphs to push company logo toward bottom
  9. Company logo: right-aligned, ~1.5 inches width
- Add section break for body content: `doc.add_section(WD_SECTION_START.NEW_PAGE)`

**Helper function: `_setup_headers_footers(doc, config)`**
- Per D-18, Pattern 7, Pitfall 2.
- For the BODY section (doc.sections[1] — the section after cover page break):
  - Header: set `header.is_linked_to_previous = False` (CRITICAL per Pitfall 2)
  - Add company logo to header paragraph, left-aligned, width=Inches(1.0)
  - Footer: set `footer.is_linked_to_previous = False`
  - Add centered page number field. Use XML field code: `parse_xml(f'<w:fldSimple {nsdecls("w")} w:instr="PAGE \\* MERGEFORMAT"/>')`
  - Set footer paragraph alignment to CENTER

**Helper function: `_parse_and_add_prose(doc, prose_text, primary_color, accent_color, code_style)`**
- Parses the generated prose text and adds content to the document
- Detects TABLE: markers and creates Word tables with branded formatting:
  - Parse pipe-delimited rows
  - Header row: white text on primary_color background (via _set_cell_background)
  - Body rows: alternating white and light accent tint. Light tint = take accent_color RGB values and mix 85% with white: `tint_r = int(r * 0.15 + 255 * 0.85)` (same for g, b). Convert to hex string.
  - Table width = full page width (autofit)
- Detects CODE_BLOCK: markers (or triple-backtick fenced blocks) and creates code-styled paragraphs:
  - Apply 'Code' style
  - Apply _set_paragraph_shading with "F2F2F2" (light grey)
  - Apply _add_paragraph_border with "D0D0D0"
- Regular text: add as Normal paragraphs
- Empty lines between paragraphs: skip (do not create blank paragraphs for every empty line)

**Main public function: `build_docx(sections: dict, config: dict) -> str`**
- `sections` = output from content_generator.generate_all_sections
- `config` = the JSON config dict from SKILL.md (with keys per D-22)
- Creates a new Document()
- Calls _create_code_style(doc)
- Extracts primary_color and accent_color from config
- Calls _build_cover_page
- Calls _setup_headers_footers
- For each section in the DEFINED ORDER (overview, sources, dataflows, mquery, datamodel, maintenance):
  - Skip if section_id not in sections dict
  - Add section heading with _add_colored_heading (level 1, primary color)
  - Call _parse_and_add_prose with the section's prose text
- Construct output filename per OUT-02 convention: `{client_name}_{report_name}_v{version}_{YYYY-MM-DD}.docx` with spaces replaced by underscores
- Ensure output_dir exists
- Save document to `os.path.join(config["output_dir"], filename)`
- Returns the full absolute path to the saved file
- Prints to stderr: f"Document saved: {output_path} ({os.path.getsize(output_path)} bytes)"
