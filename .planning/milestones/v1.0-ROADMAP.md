# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-04-02
**Phases:** 1–3

## Overview

DocsGen v1.0 delivers the complete core pipeline: a Claude Code skill that takes Power BI Markdown documentation from `pbi:docs`, runs a branded intake wizard, and produces polished `.docx` and `.pdf` deliverables in English or French — in a single invocation. All 31 v1 requirements shipped.

## Milestone Summary

**Key Decisions:**

- DOCX before PDF — validates content pipeline with fewer moving parts (no Playwright dependency)
- English before French — EN is simpler case; FR glossary pattern designed in but implemented Phase 3
- python-docx run-level formatting (not style mutation) to avoid silent heading style failures
- Keyword matching uses first-match priority from YAML ordering
- Fowler rules included via Jinja2 include for DRY compliance across all section prompts
- TABLE: and CODE_BLOCK: markers as conventions for DOCX builder to parse structured content
- SDT XML TOC field injection pattern for Word TOC (python-docx issue #36)
- CSS custom properties (--primary/--accent) for brand colors in HTML template
- pdf_builder import inside try/except for graceful degradation without Playwright
- Dual stdout protocol (docx path + pdf path or PDF_SKIPPED) for machine-readable SKILL.md capture
- Dual template map pattern (SECTION_TEMPLATE_MAP + SECTION_TEMPLATE_MAP_FR) for clean EN/FR separation
- Lazy-init module-level cache for FR glossary to avoid per-render disk reads
- Shared language helpers (format_date, get_section_label, COVER_BOILERPLATE) in docx_builder.py; pdf_builder imports for DRY
- FR date format "le 2 avril 2026" via month-name dict (locale.setlocale() is thread-unsafe on Windows)

**Issues Resolved:**

- python-docx heading style mutations fail silently — resolved with run-level RGBColor formatting from Phase 1
- Logo placement in .docx header improved during Phase 2 polish
- French PBI terminology glossary compiled from domain research during Phase 3
- SKILL.md FR gate removed after FR was fully implemented in Phase 3

**Technical Debt Incurred:**

- Parallel _prose_to_html() in pdf_builder rather than shared abstraction (to avoid refactor risk during Phase 2)
- Logo placement noted for potential improvement in a future polish phase
