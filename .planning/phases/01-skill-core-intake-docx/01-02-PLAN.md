**YAML Frontmatter** (per D-20, SCAF-03) — the file MUST begin with YAML frontmatter delimiters containing these three fields:
  - name: pbi-docgen
  - description: "Generate branded Word documents from PBI documentation. Use when the user wants to create client-ready .docx deliverables from pbi:docs Markdown output."
  - allowed-tools: Bash, Read, Write, Glob

**Body content** — a detailed markdown prompt that instructs Claude to execute the following steps in order:

**Step 0: Dependency Check (SCAF-04)**
Before anything else, run a Python one-liner via Bash to check imports:
`python -c "import docx; print('python-docx', docx.__version__)"`
If this fails, show the user:
"DocsGen requires python-docx. Install it with: pip install python-docx==1.2.0"
Stop and do not proceed until the user confirms installation.

Also verify Jinja2 and PyYAML are available:
`python -c "import jinja2; import yaml; print('OK')"`

**Step 1: Asset Directory Setup (SCAF-02, D-05, D-09)**
Check if `docsgen-assets/` exists in the current working directory.
If not, create it: `mkdir -p docsgen-assets/logos docsgen-assets/source`
Tell the user: "I've created docsgen-assets/ with logos/ and source/ subdirectories. Please place your source Markdown files in docsgen-assets/source/ and logo images in docsgen-assets/logos/."

If it already exists, check what's inside with `ls docsgen-assets/logos/ docsgen-assets/source/`.

**Step 2: Intake Wizard (INTAK-01, INTAK-02, D-06)**
Collect these 10 fields ONE AT A TIME in this order. Ask for each field, wait for the user's response, validate it, then move to the next. Do NOT ask multiple fields at once.

1. **Source MD file(s)** — Ask which .md files in docsgen-assets/source/ to use. List available files with `ls docsgen-assets/source/*.md`. If none exist, tell the user to place their pbi:docs output files there first. Accept one or more filenames.

2. **Client name** — The client organization name (e.g., "Acme Corporation"). Used on cover page and throughout the document.

3. **Client logo** — Path to client logo image in docsgen-assets/logos/. List available images with `ls docsgen-assets/logos/`. Validate the file exists. Store as absolute path.

4. **Company logo** — Path to company (your) logo image in docsgen-assets/logos/. Same validation as client logo. Store as absolute path.

5. **Primary color** — Hex color code (e.g., "#1B365D"). Used for headings and table headers. Validate it matches the pattern for 6 hex digits with optional # prefix. Ensure '#' prefix is present.

6. **Accent color** — Hex color code (e.g., "#4A90D9"). Used for secondary elements and table accents. Same validation.

7. **Language** — EN or FR. ALWAYS ask this, never cache or infer (per D-07 / INTAK-02). For Phase 1, if user selects FR, warn: "French language support is planned for Phase 3. Proceeding with English for now."

8. **Audience type** — One of: "client", "internal", "IT". Explain the difference: client = plain-English summaries, internal/IT = technical detail with annotated code blocks.

9. **Report name** — The document title (e.g., "Sales Dashboard Documentation").

10. **Version number** — Version string (e.g., "1.0", "2.1"). Default to "1.0" if user presses enter.

**Step 3: Pre-Generation Validation (INTAK-03, D-08)**
Before launching generation, validate ALL of:
- All source MD files exist and are readable (Read each to confirm)
- Client logo file exists
- Company logo file exists
- Primary color and accent color are valid hex
- Language is "EN" or "FR"
- Audience is "client", "internal", or "IT"

If anything is missing, show a checklist of all issues at once (not one by one). Do NOT proceed until all items pass.

**Step 4: Output Directory Setup (INTAK-04)**
Create docsgen-output/ in the current working directory if it does not exist: `mkdir -p docsgen-output`

**Step 5: JSON Config and Generation (D-20, D-22)**
Write a JSON config file with this exact schema (11 keys):
- "client_name": string (collected from intake)
- "client_logo": string (absolute path)
- "company_logo": string (absolute path)
- "primary_color": string (hex with # prefix, e.g. "#1B365D")
- "accent_color": string (hex with # prefix)
- "language": string ("EN" or "FR")
- "audience": string ("client", "internal", or "IT")
- "report_name": string (collected from intake)
- "version": string (collected from intake)
- "source_files": array of strings (absolute paths to source MD files)
- "output_dir": string (absolute path to docsgen-output/)

All paths MUST be absolute (per Pitfall 6). Resolve relative paths against the current working directory before writing.

Write this to `docsgen-output/.docsgen-config.json` via the Write tool.

Then invoke generation via Bash:
`cd <current_working_dir> && python "${CLAUDE_SKILL_DIR}/scripts/generate.py" "docsgen-output/.docsgen-config.json"`

Use `${CLAUDE_SKILL_DIR}` to reference the skill's own directory (this is substituted by Claude Code at runtime).

**Step 6: Report Completion**
After generate.py completes, list the output files: `ls -la docsgen-output/`
Report the output file path and size to the user.
