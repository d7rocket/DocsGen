**fowler_rules.j2** — Shared include with Fowler compliance rules:
```jinja2
**Writing quality rules (Fowler's Modern English Usage):**
1. Prefer active voice. "The report filters data" not "Data is filtered by the report."
2. No nominalization padding. "Using" not "the utilization of." "Analyzing" not "the analysis of."
3. No corporate filler. Never use: leverage, synergy, optimize, utilize, facilitate, streamline, robust, scalable, holistic, best-in-class, cutting-edge, world-class.
4. Keep sentences under 25 words. Split long sentences.
5. Be direct. State what something does, not what it "aims to" or "seeks to" do.
6. Prefer concrete nouns over abstract ones. "The table stores customer records" not "The data storage mechanism maintains entity information."
7. Use plain connectives. "because" not "due to the fact that." "so" not "as a consequence of."

These rules are MANDATORY. Violating any rule makes the output unusable.
```

**Imports:**
```python
import os
import subprocess
import sys
from jinja2 import Environment, FileSystemLoader
```

**Constants:**
```python
TEMPLATE_DIR = os.path.join(os.path.dirname(__file__), '..', 'templates', 'prompts')
SECTION_TEMPLATE_MAP = {
    'overview': 'section_overview.j2',
    'sources': 'section_sources.j2',
    'dataflows': 'section_dataflows.j2',
    'mquery': 'section_mquery.j2',
    'datamodel': 'section_datamodel.j2',
    'maintenance': 'section_maintenance.j2',
}
```

**Function 1: `_render_prompt(section_id, source_content, client_name, report_name, audience) -> str`**
- Creates Jinja2 Environment with FileSystemLoader pointing to TEMPLATE_DIR
- Loads the template for the given section_id from SECTION_TEMPLATE_MAP
- Renders with variables: `client_name`, `report_name`, `source_content`, `audience`
- Returns the rendered prompt string
- Raises KeyError if section_id is not in SECTION_TEMPLATE_MAP

**Function 2: `_call_claude(prompt_text: str) -> str`**
- Calls `claude -p` with the prompt text as a subprocess (per D-01, Pattern 8)
- Uses `subprocess.run()` with:
  - `["claude", "-p", prompt_text, "--output-format", "text"]`
  - `capture_output=True`
  - `text=True`
  - `encoding='utf-8'`
  - `errors='replace'` (per Pitfall 4: Windows encoding)
  - `timeout=120`
- If returncode != 0, raises RuntimeError with stderr content
- Returns stdout.strip()
- Prints progress to stderr: `f"  Generating {section_id}..."` before call (use a section_id param or print from caller)

**Function 3: `generate_all_sections(parsed_sections: dict, client_name: str, report_name: str, audience: str) -> dict[str, dict]`**
- Main public function
- Takes parsed_sections dict from md_parser.parse_markdown_sources (keys = section IDs, values = {"label": str, "content": str})
- For each section in parsed_sections (per D-04, each section is a separate LLM call):
  1. Render the Jinja2 prompt with _render_prompt
  2. Call _call_claude with the rendered prompt
  3. Store result
- Returns dict keyed by section ID: `{"overview": {"label": "Project Overview", "prose": "generated text..."}, ...}`
- Prints progress to stderr: `f"Generating section {i}/{total}: {label}..."` for each section
- Prints `"Content generation complete."` to stderr when done
- Skips any section_id not in SECTION_TEMPLATE_MAP (logs warning to stderr)

**Error handling:**
- If claude -p fails for one section, catch the RuntimeError, print warning to stderr, and continue with remaining sections. Set prose to empty string for the failed section.
- If TEMPLATE_DIR does not exist, raise FileNotFoundError with clear message.

**Type hints** on all functions. **Docstrings** on all functions.
