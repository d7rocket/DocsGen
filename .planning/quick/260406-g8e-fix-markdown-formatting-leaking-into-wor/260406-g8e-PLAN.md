Add a new function `_add_inline_md_paragraph(doc, line, code_style=None)` to docx_builder.py, placed BEFORE `_parse_and_add_prose`. This function:

1. Handles leading `- ` bullet indicator: strip the dash, add paragraph with 'List Bullet' style if available, else normal paragraph with the dash stripped.

2. Parses the line text into runs using a regex-based approach that handles these inline Markdown patterns in priority order:
   - `***text***` or `___text___` --> bold + italic run
   - `**text**` or `__text__` --> bold run
   - `*text*` or `_text_` --> italic run (but NOT mid-word underscores like `Sales_Amount` -- only match `_text_` when preceded by whitespace or start-of-string and followed by whitespace or end-of-string/punctuation)
   - `` `text` `` --> Courier New 9pt run (reuse code_style font settings: Courier New, 9pt, color #333333, no background shading)
   - Plain text --> normal run

3. Uses a single regex pattern to split the line into segments:
   ```python
   INLINE_MD_RE = re.compile(
       r'(\*\*\*(.+?)\*\*\*'    # ***bold+italic***
       r'|___(.+?)___'           # ___bold+italic___
       r'|\*\*(.+?)\*\*'        # **bold**
       r'|__(.+?)__'            # __bold__
       r'|\*(.+?)\*'            # *italic*
       r'|(?<=\s)_(.+?)_(?=[\s.,;:!?\)]|$)'  # _italic_ (word-boundary aware)
       r'|`([^`]+)`'            # `code`
       r')'
   )
   ```

4. Iterates through matches using `re.finditer`, collecting plain text between matches as normal runs, and creating appropriately formatted runs for each match group.

5. Returns the created paragraph.
